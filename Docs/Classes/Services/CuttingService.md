# CuttingService

**Namespace:** `LinearCutWpf.Services`
**Тип:** `class`

## Назначение
Класс управляет бизнес-логикой подготовки данных перед отправкой в математическое ядро раскроя (`CutOptimizer`). Служит "прослойкой" между UI (вводом пользователя из таблиц и настроек) и чистыми алгоритмами.

Главная задача класса — правильно обработать **ручной раскрой** (детали, которые пользователь решил распилить вручную), вычесть их из общего пула деталей, переиспользовать остатки от ручного раскроя, и запустить оптимизацию по каждой группе (артикулу) отдельно.

## Зависимости
- `LinearCutWpf.Models` (`PresetModel`, `StockLengthModel`, `ManualCutRow`, `CutBar`, `PartItem`, `CutBarDetailed`).
- `System.Data` (используется `DataTable`, `DataRow` для работы с загруженными из Excel данными и генерации таблиц результатов).
- `LinearCutWpf.Services.CutOptimizer` (используется для непосредственно математического расчета раскроя).

## Вложенные классы (Модели данных)
1. **`ManualValidationError`**: Содержит информацию об ошибке валидации конкретной ячейки в таблице ручного раскроя (индекс строки, имя колонки, сообщение).
2. **`OptimizationResult`**: Агрегирует итоговый результат по одному артикулу. Содержит:
   - Исходный `DataTable` для DataGrid (`ResultTable`).
   - Статистику: КПД, общее кол-во деталей, общая длина хлыстов, общая длина остатков.
   - Детализированный список резов `List<CutBarDetailed>`.
3. **`GroupData`**: Входная структура данных (строки из Excel, сгруппированные по артикулу).

## Основные методы

### Валидация
- **`ValidateManualCuts`**: Проходится по строкам ручного раскроя (`ManualCutRow`). Вычисляет эффективную длину хлыста (`длина - торцовка - припуски`). Суммирует длины введенных деталей (включая ширину реза). Если детали не помещаются в хлыст, генерирует `ManualValidationError`.
- **`VerifyOptimization`**: Метод самопроверки (используется в основном в тестах, но вызывается и в MainWindow). Сверяет сумму длин всех исходных деталей из `GroupData` с суммой длин всех деталей в итоговом `OptimizationResult`. Если они расходятся, значит алгоритм где-то "потерял" или "выдумал" деталь.

### Подготовка и Запуск раскроя
- **`OptimizeAllGroups`**: Центральный метод. Алгоритм работы:
  1. Вызывает `BuildManualParts` для получения плоского списка длин всех деталей, распиленных вручную.
  2. Если есть ручной раскрой, и установлена галочка `UseRemainders` — вычисляет длину обрезков ручного раскроя и добавляет их в список `finiteStocks` (хлысты ограниченного количества).
  3. Проходит в цикле по каждой группе (артикулу) `GroupData`:
     - Читает `DataTable` из Excel.
     - Для каждой детали проверяет: если она уже есть в списке `manualParts` (с учетом погрешности `Math.Abs < 0.1`), то деталь удаляется из списка `manualParts` и *не добавляется* в список на автоматический раскрой.
     - Формирует список `PartItem` и отправляет его в `CutOptimizer.Optimize`.
     - Результат от `CutOptimizer` (`List<CutBarDetailed>`) конвертирует обратно в старый формат `CutBar` для UI.
     - Собирает объект `OptimizationResult`.
  4. Если есть результаты ручного раскроя, они оформляются в отдельный `OptimizationResult` (псевдо-группа `"Ручной раскрой"`) для отображения отдельной вкладкой в UI.

### Вспомогательные методы
- **`BuildManualCutBars`**: Преобразует `ManualCutRow` в список хлыстов с остатками.
- **`BuildResultTable`**: Группирует одинаковые хлысты (через LINQ `GroupBy` по длине хлыста и карте реза) и формирует компактный `DataTable` (Кол-во | Хлыст | Раскрой | Остаток) для UI.
- **`BuildManualParts`**: Конвертирует 4 колонки из `ManualCutRow` в плоский список `double` (каждая деталь + ширина реза).

## Особенности реализации
- Связь между таблицей Excel и списком деталей осуществляется через имена колонок (`ValueColumnName`, `QtyColumnName`). Значения парсятся через `Convert.ToDouble` и `Convert.ToInt32`.
- Класс активно использует `DBNull.Value` и безопасный парсинг для работы с неидеальными данными (пустые строки в Excel).
- Ручной раскрой жестко ограничен 4-мя колонками (`Size1`..`Size4`). Это "наследие", которое можно масштабировать при необходимости.
