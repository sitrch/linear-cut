# CutOptimizer

## Назначение
Статический класс `CutOptimizer` предоставляет алгоритмы для оптимизации линейного раскроя хлыстов (погонажных изделий). Он реализует "жадный" алгоритм (First Fit Decreasing - FFD) для распределения списка деталей по доступным хлыстам с минимизацией остатков.

## Зависимости (DI)
Класс статический, не использует внедрение зависимостей. Зависит только от моделей из пространства имен `LinearCutWpf.Models` (`CutBar`, `CutBarDetailed`, `PartItem`).

## Подробный алгоритм работы методов

### 1. `Optimize(List<double> parts, List<double> stocks, double reduction, double cutWidth)`
**Назначение:** Базовый метод оптимизации раскроя (упрощенный вариант, оперирует только длинами типа `double`).
**Алгоритм:**
1. Инициализирует результирующий список `CutBar`.
2. Сортирует входной список деталей `parts` по убыванию длины (`remainingParts`).
3. В цикле `while`, пока есть неразмещенные детали (`remainingParts.Any()`):
   - Ищет первый подходящий хлыст (`bestStock`) из отсортированного по убыванию списка `stocks`, полезная длина которого (`s - reduction`) больше или равна самой длинной из оставшихся деталей.
   - Если хлыст не найден, удаляет эту деталь (она не поместится ни в один хлыст) и переходит к следующей итерации.
   - Вычисляет `capacity` (полезную длину хлыста) как `bestStock - reduction`.
   - Инициализирует список деталей для текущего хлыста (`currentBarParts`).
   - Если самая длинная деталь занимает больше половины хлыста (`> capacity / 2`), сразу добавляет ее в хлыст (оптимизация).
   - Проходит по остальным деталям в цикле: если текущая деталь помещается в остаток хлыста (`capacity - currentBarParts.Sum()`), добавляет ее в хлыст и удаляет из списка неразмещенных.
   - Создает объект `CutBar`, записывая длину хлыста, строковое представление деталей (с вычетом ширины реза `cutWidth` для отображения чистого размера) и рассчитывает остаток. Добавляет его в результаты.
4. Возвращает список `CutBar`.

### 2. `Optimize(List<double> parts, List<double> infiniteStocks, List<double> finiteStocks, double tStart, double tEnd, double cWidth)`
**Назначение:** Расширенный метод оптимизации с учетом конечных и бесконечных остатков. Оперирует длинами `double`.
**Алгоритм:**
1. Вычисляет общий торцевой припуск `reduction` с учетом ширины реза: `(tStart - cWidth / 2) + (tEnd - cWidth / 2)`.
2. Подготавливает список конечных остатков `availableFiniteStocks`, сортируя их по убыванию (или создает пустой список, если передан `null`).
3. В цикле по оставшимся деталям (`remaining`):
   - **Сначала пытается использовать конечные остатки.** Проходит по `availableFiniteStocks`. Если остаток (который считается "чистой вместимостью" без припусков) больше или равен самой длинной детали, выбирает его (`useFinite = true`).
   - Если конечный остаток не подошел (`!useFinite`), ищет подходящий хлыст среди `infiniteStocks` (отсортированных по убыванию), учитывая `reduction`.
   - Если подходящий хлыст/остаток не найден, пропускает самую длинную деталь.
   - Если использовался конечный остаток, удаляет его из списка доступных `availableFiniteStocks`.
   - Далее логика заполнения хлыста (FFD) аналогична базовому методу: жадно добавляет детали, пока они помещаются в текущую `capacity`.
   - Формирует и сохраняет объект `CutBar`.

### 3. `Optimize(List<double> parts, List<double> stocks, double tStart, double tEnd, double cWidth)`
**Назначение:** Перегрузка расширенного метода оптимизации без использования конечных остатков (передает `null` вместо `finiteStocks`).

### 4. `Optimize(List<PartItem> parts, List<double> infiniteStocks, List<double> finiteStocks, double tStart, double tEnd, double cWidth)`
**Назначение:** Детализированная оптимизация раскроя, возвращающая объекты `CutBarDetailed`. В отличие от предыдущих методов, работает со списком объектов `PartItem` (деталей), что позволяет сохранить дополнительную информацию (название, цвет и т.д.).
**Алгоритм:**
- Логика выбора хлыстов (сначала конечные `finiteStocks`, затем бесконечные `infiniteStocks`) и алгоритм FFD (жадное заполнение) полностью идентичны методу 2.
- Главное отличие: вместо сохранения только списка длин деталей в виде строки, метод формирует список оригинальных объектов `PartItem` для каждого реза и сохраняет их в свойство `Parts` объекта `CutBarDetailed`. Длина деталей используется из свойства `PartItem.Length`.
- Возвращает список детализированных раскройных карт `CutBarDetailed`.

## Связи с другими классами
- Используется в `CuttingService` для выполнения основной бизнес-логики раскроя.
- Оперирует моделями данных `CutBar`, `CutBarDetailed`, `PartItem` из `LinearCutWpf.Models`.
