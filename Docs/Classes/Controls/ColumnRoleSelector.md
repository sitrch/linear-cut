## 1. Общее описание
**Файл:** `Controls/ColumnRoleSelector.xaml` + `Controls/ColumnRoleSelector.xaml.cs`  
**Пространство имен:** `LinearCutWpf.Controls`  
**Назначение:** Выпадающий селектор ролей столбца, встраиваемый в заголовок DataGrid. Заменяет табличный способ назначения ролей столбцов (через `dgColumnConfig`). Отображает список ролей с чекбоксами: Key, Наименование, Value, Quantity, Левый угол, Правый угол, Цвет.

## 2. Зависимости
- `System.Windows.Controls`: Базовые контролы WPF (`UserControl`, `ToggleButton`, `Popup`, `CheckBox`, `StackPanel`).
- Не зависит от моделей или сервисов — чистый UI-контрол.

## 3. Свойства и Поля
- **DependencyProperty `DisplayText`**: Текст, отображаемый в заголовке селектора. Привязан к `Content` ToggleButton. По умолчанию — "Выбрать...".
- **`ColumnName`**: Имя столбца, к которому привязан селектор. Устанавливается извне (`DataSettingsControl`).
- **`Dictionary<string, CheckBox> _roleCheckboxes`**: Внутреннее хранилище чекбоксов по ключам ролей (`IsKey`, `IsName`, `IsVal`, `IsQty`, `IsLeftAngle`, `IsRightAngle`, `IsColor`).
- **`HashSet<CheckBox> _suppressEvents`**: Подавляет рекурсивные события при программном изменении чекбоксов.

## 4. События
- **`RoleChanged`**: Срабатывает при изменении выбора роли. Параметры: `ColumnName`, `RoleKey`, `IsChecked`.

## 5. Методы
- **`ColumnRoleSelector()`**: Инициализирует визуальные элементы и подписывается на события.
- **`BuildRoleItems()`**: Создает чекбоксы для каждой роли внутри Popup.
- **`SetRoleChecked(string roleKey, bool isChecked)`**: Программная установка состояния роли (без вызова события).
- **`GetCheckedRole()`**: Возвращает первую выбранную роль или null.
- **`UpdateDisplayText()`**: Обновляет текст заголовка селектора на имя первой выбранной роли.
- **`OnRoleChecked / OnRoleUnchecked`**: Обработчики изменения чекбоксов. Реализуют RadioButton-поведение: при выборе одной роли снимаются остальные. После изменения чекбокса Popup автоматически закрывается (`popup.IsOpen = false`).

## 6. Особенности реализации
- **RadioButton-поведение**: В пределах одного столбца может быть выбрана только одна роль (исключение — роль `IsKey`, которая может быть назначена нескольким столбцам одновременно; контроль этого лежит в `DataSettingsControl`).
- **Popup + ToggleButton**: Вместо ComboBox используется комбинация ToggleButton + Popup для гибкого управления открытием/закрытием списка.
- **Автоматический текст заголовка**: После выбора роли заголовок отображает её название; при отмене выбора — "Выбрать...".
- **Автозакрытие Popup**: После клика по любому чекбоксу (включение или выключение) выпадающий список закрывается автоматически.
