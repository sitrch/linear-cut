# ArticleGroupingControl

**Пространство имен:** `LinearCutWpf.Controls`  
**Тип:** `UserControl`

## Назначение
Контрол `ArticleGroupingControl` отвечает за отображение и редактирование настроек конкретного артикула (группы деталей) перед запуском алгоритма раскроя. В нем можно переопределить длину хлыста и пресет обрезки для данного артикула, а также задать ручные резы, минуя автоматический оптимизатор.

## Основные элементы интерфейса
- **Левая панель:**
  - **Информация об артикуле:** Вверху панели — наименование артикула (ключ + данные из столбцов «Наименование» (IsName) и «Цвет» (IsColor)) с цветовым индикатором (зелёный — настройки по умолчанию, красный — есть индивидуальные настройки).
  - **Информационная панель (`infoLabel`):** Расположена сразу под информацией об артикуле (до блока настроек хлыста/пресета). Показывает количество деталей, общую длину, настройки хлыста и пресета. Перенесена выше блока ручного раскроя, чтобы не пропадать при его открытии.
  - Выбор хлыста и пресета, включение/выключение и отображение контрола ручного раскроя (`ManualCutControl`). Ширина панели управляется через `LeftPanelWidth` и `GridSplitter`; вложенные элементы (`manualCutFrame`, `errorsBorder`) растягиваются на всю доступную ширину.
- **Правая панель:** Таблица детализации (`DataGrid`), показывающая сгруппированные детали (наименование, длина, количество) для текущего артикула. Поддерживается произвольное выделение ячеек (как в Excel): `SelectionMode="Extended"`, `SelectionUnit="Cell"` — можно выделять отдельные ячейки с Ctrl+Click и Shift+Click.
- **Строка статуса:** Внизу правой панели отображается сумма значений выделенных ячеек в формате «Наименование столбца: ххх» (например, «Длина: 1200», «Количество: 5»). Для текстовых столбцов показывается количество выделенных ячеек.
- **Информационный блок:** Отображает общую статистику (кол-во деталей, общая длина) и текущие настройки.
- **Панель ошибок:** Выводит сообщения об ошибках валидации ручного раскроя (например, превышение длины хлыста или нехватка доступных деталей).

## Основные свойства и события
- `LeftPanelWidth` (DependencyProperty): Управляет шириной левой панели настроек.
- `ManualCutControl`: Возвращает ссылку на вложенный контрол ручного раскроя.
- `GridDetails`: Возвращает таблицу детализации.
- `event SettingsChanged`: Событие, генерируемое при изменении любых настроек артикула. Используется родительским окном/контролом для обновления визуальных индикаторов (например, перекраски вкладок).

## Ключевые методы
- `Initialize(...)`: Основной метод инициализации. Принимает данные об артикуле, доступные хлысты и пресеты, исходные строки из Excel. Вызывает построение левой и правой панелей.
- `BuildLeftPanel()`: Формирует списки хлыстов и пресетов, инициализирует `ManualCutControl` и настраивает подписки на события коллекции `ManualCuts`. Заполняет наименование артикула и цветовой индикатор в верхней части панели.
- `BuildRightPanel(...)`: Группирует входные данные (строки Excel) по уникальным комбинациям ключей, рассчитывая суммарную длину и количество деталей. Результат выводится в `GridDetails`.
- `HasManualErrors()`: Возвращает флаг наличия ошибок валидации в ручном раскрое.
- `UpdateInfoBlock()`: Рассчитывает общую статистику по артикулу и обновляет текстовый блок.
- `UpdateIndicators()`: Обновляет визуальные индикаторы (зеленые галочки/красные точки) рядом с настройками, показывая, переопределены ли они относительно глобальных или нет. Также обновляет цветовой индикатор артикула.
- `UpdateArticleColorIndicator()`: Обновляет цвет индикатора артикула (зелёный/красный) в зависимости от наличия индивидуальных настроек.
- `BuildArticleDisplayName()`: Формирует отображаемое имя артикула из ключа + значений столбцов «Наименование» (IsName) и «Цвет» (IsColor) из первой строки данных.
- `OnSelectedCellsChanged(...)`: Обработчик выделения ячеек в DataGrid. Рассчитывает сумму значений по каждому столбцу и отображает результат в строке статуса в формате «Наименование столбца: ххх».
- `RefreshManualGridValidation()`: Пробрасывает актуальные значения хлыста и пресета в `ManualCutViewModel` для пересчета доступного места и остатков.

## Взаимодействие (Dependencies)
- **Модели:** `ArticleSettings`, `StockLengthModel`, `PresetModel`, `ManualCutRow`. Используются для хранения и передачи настроек.
- **Вложенные контролы:** Строго связан с `ManualCutControl` и его ViewModel (`ManualCutViewModel`), которым передает делегированное управление ручным раскроем.
- Использует класс `DataHelper` для формирования ключа артикула.
