# Описание класса ResultsControl

## Уровень: Local (Level 2)
**Пространство имен:** `LinearCutWpf.Controls`  
**Сборка:** `LinearCutWpf`  
**Тип:** `UserControl`  

## Назначение
Контрол `ResultsControl` предназначен для отображения результатов оптимизации раскроя. Он получает список результатов, рассчитывает общую статистику по всем артикулам и формирует списки для отображения таблиц раскроя и статистики по каждой отдельной группе (артикулу).

## Зависимости (DI и связи)
- **CuttingService.OptimizationResult:** Использует результаты работы сервиса `CuttingService` для формирования отображаемых данных.

## Вложенные типы
### `ResultViewModel`
Вспомогательный класс-модель представления, используемый для привязки данных результатов по каждой отдельной группе в интерфейсе. Содержит свойства:
- `GroupKey`: Артикул или ключ группы (включает наименование и цвет, если заданы). Для ручного раскроя содержит суффикс «(Ручной раскрой)».
- `ArticleDescription`: Описание артикула (наименование и цвет из данных).
- `IsManualCut`: Флаг, указывающий, что данный результат относится к ручному раскрою. Используется для цветовой подсветки (жёлтый фон `#FFF8E1`, оранжевый текст `#E65100`).
- `ResultTable`: Таблица данных с раскроем.
- `StatsParts`, `StatsStocks`, `StatsRemainders`, `StatsKpd`: Строки форматированной статистики для отображения. Для ручного раскроя `StatsParts` содержит текст «Раскроено вручную» вместо «Оптимизировано».

## Основные элементы и методы
- **Конструктор `ResultsControl()`**: Инициализирует компоненты UI.
- **Метод `DisplayResults(List<CuttingService.OptimizationResult> results)`**: 
  - Проверяет наличие данных.
  - Рассчитывает и отображает общую статистику (общая длина деталей, хлыстов и средний КПД) в `txtTotalStats`.
  - Формирует список объектов `ResultViewModel` для каждой группы из `results`.
  - Форматирует статистику по хлыстам, деталям и остаткам.
  - Устанавливает сформированный список в качестве `ItemsSource` для `resultsItemsControl`.
