# Уровень 2: Документация класса ArticleSettingsControl

## 1. Общее описание
**Файл:** `Controls/ArticleSettingsControl.xaml.cs`  
**Пространство имен:** `LinearCutWpf.Controls`  
**Назначение:** UI-контрол (UserControl) для отображения и настройки параметров раскроя в разрезе артикулов (материалов). Позволяет пользователю задать видимую высоту профиля, переопределить длину хлыста по умолчанию и выбрать специфический пресет настроек для каждого уникального артикула.

## 2. Зависимости
- `System.Collections.ObjectModel`: Для использования `ObservableCollection`.
- `System.Data`: Для работы с `DataTable` и `DataRow`.
- `LinearCutWpf.Models`: Использует модели `ArticleGroupingRow`, `StockLengthModel`, `PresetModel`, `ArticleSettings`.
- Зависит от вспомогательного класса `DataHelper` (вызов `DataHelper.GetArticleName`).

## 3. Свойства
- `ObservableCollection<ArticleGroupingRow> Articles { get; set; }`: Коллекция строк (артикулов), привязанная к `DataGrid`. Содержит агрегированную информацию о деталях (общее количество, длина) и выбранные пользователем настройки.
- `ObservableCollection<StockLengthModel> AvailableStockLengths { get; set; }`: Коллекция доступных хлыстов.
- `ObservableCollection<PresetModel> AvailablePresets { get; set; }`: Коллекция доступных пресетов.
- `double? DefaultVisibleHeight { get; set; }`: Высота по умолчанию для артикулов.

## 4. Методы
- `ArticleSettingsControl()`: Конструктор. Инициализирует компонент и устанавливает привязку `dgArticles.ItemsSource`.
- `Initialize(...)`: Агрегирует сырые данные из `DataTable` по артикулам, используя указанные колонки, формирует список `ArticleGroupingRow`. Подмешивает к результату пустые значения (по умолчанию) для хлыстов и пресетов, а также восстанавливает ранее выбранные настройки из `existingSettings`. Загружает сохраненные данные о высотах через `ProfileHeightService`.
- `GetSettings()`: Собирает и возвращает только те настройки артикулов, которые были изменены пользователем (выбрана специфичная длина или пресет). Сохраняет данные о высотах через `ProfileHeightService`.

## 5. Особенности реализации
- Автоматически вычисляет общее количество деталей и суммарную длину для каждого артикула при инициализации.
- При формировании выпадающих списков для `DataGrid` (хлысты и пресеты) добавляется пустое значение, которое означает "использовать настройки по умолчанию".
- Если при повторном импорте или обновлении данных в `Initialize` передаются `existingSettings`, ранее заданные пользователем параметры восстанавливаются.
- Поддерживает цветовую индикацию ячеек с помощью `DataTrigger` в XAML (привязано к флагам в модели `ArticleGroupingRow`):
  - **Видимая высота**: Светло-голубой (`#E6F3FF`) — значение по умолчанию, розовый (`#FFB6C1`) — изменено вручную.
  - **Длина хлыста**: Серый (`#E6E6E6`) — значение по умолчанию (не выбрано), розовый (`#FFB6C1`) — изменено вручную, белый — выбрано из списка.
  - **Пресет**: Серый (`#E6E6E6`) — значение по умолчанию (не выбрано), белый — выбран пользователем.
- Отслеживает изменения высоты по умолчанию и автоматически обновляет соответствующие ячейки.
- Обработчики событий: `OnBarLengthCellSelectionChanged`, `OnPresetCellSelectionChanged` — отслеживают взаимодействие пользователя с ячейками и обновляют флаги (включая `IsBarLengthManuallyChanged`).
- При переключении на вкладку "Настройка артикулов" загружает сохранённые настройки (видимая высота и длина хлыста) из XML-файла через `ProfileHeightService`.
- При уходе со вкладки или закрытии приложения настройки сохраняются в XML-файл через `ProfileHeightService`.
