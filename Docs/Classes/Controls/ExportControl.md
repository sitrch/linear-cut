# Уровень 2: Документация класса ExportControl

## 1. Общее описание
Класс `ExportControl` (наследует `UserControl`) представляет собой пользовательский интерфейс для экспорта результатов оптимизации раскроя. Он поддерживает генерацию отчетов в форматах Excel и PDF. Предоставляется выбор из трех типов отчетов: "Подробный отчёт", "Отчёт об использовании материалов" и "Визуальный отчёт" (в PDF с графической схемой раскроя).

## 2. Зона ответственности
- Отображение списка артикулов и результатов раскроя для выбора пользователем (флажки выбора).
- Формирование и экспорт Excel-отчетов (с использованием библиотеки ClosedXML), включающих сводную таблицу и детальную спецификацию.
- Формирование и экспорт PDF-отчетов (с использованием библиотеки QuestPDF), включая детальный, материальный и визуальный (с отрисовкой SVG) форматы.
- Отрисовка визуальной схемы (SVG) хлыстов и деталей (с учетом углов реза) для отчета "Визуальный отчёт".

## 3. Зависимости (DI и связи)
- `LinearCutWpf.Services.CuttingService` — для доступа к структурам данных `OptimizationResult` и деталям раскроя.
- `LinearCutWpf.Models.CutBarDetailed` — для визуального отображения детализированных хлыстов.
- `LinearCutWpf.Services.CutSettingsProvider` — для сохранения и загрузки коэффициента высоты визуального профиля.
- `LinearCutWpf.Services.ProfileHeightService` — для получения высоты профиля артикула при генерации визуального отчёта.
- `QuestPDF.Fluent`, `QuestPDF.Helpers`, `QuestPDF.Infrastructure` — для генерации PDF-файлов.
- `ClosedXML.Excel` — для формирования Excel-файлов.
- `DataHelper` — для формирования ключа артикула из набора ключевых столбцов.

## 4. Вложенные классы
### `ExportRowModel`
Модель данных для строки в таблице экспорта. Реализует `INotifyPropertyChanged`.
- `IsSelected` — флаг выбора строки для экспорта (по умолчанию `true`).
- `Article` — артикул (ключ группы).
- `Name` — наименование артикула из исходных данных.
- `Color` — цвет артикула из исходных данных (из столбца с ролью `IsColor`).
- `StockLength` — длина основного хлыста.
- `StockCount` — общее количество хлыстов.
- `IsManualCut` — флаг ручного раскроя (используется для цветовой подсветки ячеек).
- `ResultData` — ссылка на `OptimizationResult` для выгрузки.

## 5. Основные методы
- `ExportControl()` — конструктор. Восстанавливает значение коэффициента высоты визуального отчёта из настроек.
- `LoadData(List<CuttingService.OptimizationResult>, DataTable, ..., string colorColumnName = null)` — загружает результаты раскроя и оригинальную таблицу с данными для привязки к интерфейсу. Извлекает наименование и цвет артикула из первой совпадающей строки исходной таблицы. Параметр `colorColumnName` опциональный — имя столбца с ролью `IsColor`.
- `BtnExportExcel_Click` — формирует Excel-файл со сводной и детальной статистикой.
- `BtnExportPdf_Click` — инициирует экспорт в PDF. Выбирает метод генерации в зависимости от типа отчета (`cbReportTypePdf.SelectedIndex`).
- `ExportVisualPdf` / `ComposeVisualContent` / `ComposeVisualArticle` — методы для сборки структуры визуального PDF.
- `GenerateVisualBarSvg` — генерирует SVG-представление одного хлыста. Учитывает реальную высоту профиля и пропорциональную длину, использует коэффициент масштабирования высоты, а также учитывает толщину реза и углы.
- `txtVisualHeightCoef_TextChanged` — обрабатывает изменение пользователем коэффициента масштабирования высоты и сохраняет его в настройки.
- `chkSelectAll_Click` / `dgExportData_PreviewKeyDown` — вспомогательные методы для управления выбором элементов в таблице.

## 6. Визуализация таблицы экспорта
- **Столбцы DataGrid:** Сохранить (checkbox), Артикул, Наименование, Цвет, Длина хлыста, Количество хлыстов.
- **Подсветка ручного раскроя:** Цветовое выделение `#FFF8E1` (жёлтый фон) применяется только к ячейкам с данными (столбцы Артикул, Наименование, Цвет, Длина хлыста, Количество хлыстов) через `ElementStyle` с `DataTrigger` на `IsManualCut`. Столбец с чекбоксом не подсвечивается.

## 6.1. Структура экспортируемых отчётов
### Сводная таблица (Excel — лист «Сводная таблица», PDF — все типы)
- **Заголовок:** Объект (опционально), Дата формирования.
- **Столбцы:** Артикул | Наименование | Цвет | Хлыст | Кол-во | Длина (м) | %ИспМат.
- **Длина (м):** Вычисляется как `Хлыст × Кол-во / 1000`.
- **Блок статистики:** Всего деталей, Общая длина деталей/хлыстов/остатков (м), %ИспМат.

### Детальный раскрой (Excel — лист «Детальный раскрой», PDF — подробный отчёт)
- **Заголовок артикула:** Формат `Артикул: XXX | Наименование | Цвет` (одна строка, цвет опционально).
- **Таблица раскроя:** Хлыст | Кол-во | Раскрой | Остаток.
- **Блок статистики** по каждому артикулу.

### Визуальный отчёт (PDF)
- **Заголовок артикула:** Формат `Артикул: XXX | Наименование | Цвет` (одна строка, цвет опционально).
- **Перечень заготовок:** Длина | Угол левый | Угол правый | Кол-во.
- **Графическая схема** хлыстов (SVG).

## 7. Важные замечания
- Контрол сильно зависит от сторонних библиотек: `ClosedXML` и `QuestPDF`.
- Генерация графики SVG (`GenerateVisualBarSvg`) выполняется вручную с помощью строкового конструирования, чтобы получить компактный размер PDF и точный контроль над отрисовкой.
- Штриховка (фоновая заливка) должна служить подложкой и отрисовываться на всю длину хлыста.
- В PDF-отчетах (визуальном) реализована защита от разрыва элементов (`EnsureSpace`), чтобы хлысты не переносились частями на разные страницы.
