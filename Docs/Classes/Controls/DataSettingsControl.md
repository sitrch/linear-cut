## 1. Общее описание
**Файл:** `Controls/DataSettingsControl.xaml.cs`  
**Пространство имен:** `LinearCutWpf.Controls`  
**Назначение:** Главный UI-контрол первого этапа ("Загрузка данных"). Отвечает за:
1. Импорт данных из файлов Excel (`.xlsx`).
2. Настройку ролей столбцов через выпадающие селекторы в заголовках таблицы данных (Ключ/Артикул, Наименование, Длина, Количество, Левый угол, Правый угол, Цвет).
3. Валидацию загруженных данных.
4. Управление глобальными списками длин хлыстов и пресетов настроек обрезки.
5. Выбор глобальных значений хлыста и пресета по умолчанию.

### Изменения
- **Удален `dgColumnConfig`:** Таблица настройки ролей столбцов заменена на выпадающие селекторы (`ColumnRoleSelector`), встроенные непосредственно в заголовки столбцов `dgInput`.
- **Роль "Цвет" (IsColor):** Сохранена, доступна через селекторы заголовков.
- **Прямое автозаполнение ключей:** Автозаполнение пустых значений ключевых столбцов по-прежнему происходит непосредственно в `_mainDataTable` через `DataStoreService`.

## 2. Зависимости
- `ClosedXML.Excel`: Для чтения данных из `.xlsx` файлов.
- `System.Data`: Для работы с `DataTable`, которая является основой хранения импортированных данных.
- `LinearCutWpf.Models`: Использует `PresetModel`, `StockLengthModel`, `ArticleSettings`.
- `LinearCutWpf.Services`: Вызывает `CutSettingsProvider` для загрузки и сохранения пресетов, хлыстов и других пользовательских настроек; `DataStoreService` — единый источник данных.
- `LinearCutWpf.Controls`: Вызывает `ValidationDialog` для отображения ошибок валидации; использует `ColumnRoleSelector` для назначения ролей столбцов.

## 3. Свойства и Поля
- **Основные данные:**
  - `DataTable MainDataTable`: Хранит "сырые" загруженные данные. **Важно:** Автозаполнение пустых значений ключевых столбцов выполняется непосредственно в этой таблице, чтобы строки корректно попадали в группировку.
  - `DataTable ColumnConfigTable`: Хранит настройки ролей для каждого столбца (флаги `IsKey`, `IsName`, `IsVal`, `IsQty`, `IsLeftAngle`, `IsRightAngle`, `IsColor`).
  - `Dictionary<string, ColumnRoleSelector> _columnRoleSelectors`: Коллекция селекторов ролей, индексированная по имени столбца.
- **Глобальные настройки:**
  - `List<PresetModel> Presets`: Текущий список всех доступных пресетов.
  - `List<StockLengthModel> StockLengths`: Текущий список всех доступных хлыстов.
  - `double DefaultBarLength`, `PresetModel DefaultPreset`: Выбранные значения по умолчанию.
  - `string ObjectName`: Название объекта/проекта (текстовое поле).
- **События:**
  - `DataLoaded`: Сигнализирует о завершении загрузки файла.
  - `SettingsChanged`: Сигнализирует об изменении конфигурации столбцов или дефолтных настроек.

## 4. Методы
- `DataSettingsControl()`: Инициализация и вызов `LoadInitialData()`.
- `LoadInitialData()`: Загружает списки пресетов и хлыстов из `CutSettingsProvider`.
- **Импорт Excel:**
  - `OnOpenFileClick`: Открывает диалог выбора файла, считывает список листов.
  - `LoadDataFromSheet`: Читает выбранный лист через ClosedXML, формирует `_mainDataTable` и инициализирует `_columnConfigTable`.
- **Логика данных и UI:**
    - `OnDgInputAutoGeneratingColumn`: Для каждого автоматически генерируемого столбца создает `ColumnRoleSelector`, привязывает к событию `RoleChanged` и устанавливает как заголовок столбца. Служебный столбец `_ArticleKey_` отменяет генерацию (`e.Cancel = true`) и не отображается в UI.
- `OnDgInputLoaded`: Инициализирует состояние селекторов из `_columnConfigTable` и применяет визуальную раскраску столбцов.
- `OnProcessedDataChanged`: Обновляет `ItemsSource` и восстанавливает состояние селекторов и раскраску заголовков после пересоздания колонок.
    - `InitializeColumnRoleSelectors()`: Устанавливает выбранные роли в селекторах на основе текущей `_columnConfigTable`.
    - `OnColumnRoleChanged`: Обработчик изменения роли через селектор. Обновляет `_columnConfigTable`, реализует RadioButton-поведение между столбцами (кроме `IsKey`), вызывает `_dataStore.UpdateColumnConfig()` и `RefreshColumnsVisuals()`.
    - `ProcessDataLogic()`: Обрабатывает `_mainDataTable` перед отображением (замена точек на запятые, удаление пустых строк, автозаполнение ключей).
    - `GetCheckedCols(string colType)`: Вспомогательный метод для получения списка столбцов, отмеченных определенной ролью.
- `RefreshColumnsVisuals()`: Применяет цветовую раскраску к заголовкам `dgInput` через `HeaderStyle` с `BasedOn` базового стиля из XAML. Цвета: IsKey — LightGreen, IsName — LightPink, IsVal — LightYellow, IsQty — LightCyan, IsLeftAngle — LightGray, IsRightAngle — LightGray, IsColor — LightSalmon.
- **Управление настройками (CRUD):**
  - `OnAddStock`, `OnRemoveStock`: Добавление и удаление длины хлыста.
  - `OnAddPreset`, `EditPreset`, `DeletePreset`: Управление пресетами через диалоговые окна (Window).
- **Валидация:**
  - `ValidateData()`: Проверяет, что все столбцы с ролью "Длина" и "Количество" содержат корректные положительные числа. Если есть ошибки, показывает `ValidationDialog`.
- `LoadSettings(...)`: Метод для восстановления состояния контрола (используется при переключении вкладок в `MainWindow`).
- `UpdateArticleSettings(...)`: Обновляет локальные дефолтные значения, если они были изменены на других вкладках.

## 5. Особенности реализации
- **Визуализация:** Цветовая раскраска столбцов `dgInput` и их заголовков:
  - `IsKey` — LightGreen (светло-зеленый)
  - `IsName` — LightPink (светло-розовый)
  - `IsVal` — LightYellow (светло-желтый)
  - `IsQty` — LightCyan (светло-голубой)
  - `IsLeftAngle` — LightGray (светло-серый)
  - `IsRightAngle` — LightGray (светло-серый)
  - `IsColor` — LightSalmon (светло-лососевый)
- **Радио-кнопка между столбцами:** Метод `OnColumnRoleChanged` гарантирует, что одна роль (кроме `IsKey`) может быть назначена только одному столбцу. Если роль назначается новому столбцу — она автоматически снимается с предыдущего.
- **Роль Key (IsKey) множественная:** Роль `IsKey` может быть назначена нескольким столбцам одновременно (например, артикул + доп. идентификатор).
- **Автозаполнение (Merge Cells):** Если в файле Excel были объединены ячейки, `DataStoreService` автоматически копирует значение ключа из предыдущей строки. Каждая конкретная ячейка, значение которой было автозаполнено, окрашивается красным шрифтом (`Foreground = Brushes.Red`), а не вся строка целиком.
- **Визуализация автозаполнения:** В `OnDgInputLoadingRow` через `Dispatcher.BeginInvoke` с приоритетом `Background` находится `DataGridCellsPresenter` строки. Для каждой колонки, имя которой присутствует в `_dataStore.AutoFilledKeys` для данной строки, соответствующая ячейка визуального дерева получает красный цвет текста.
