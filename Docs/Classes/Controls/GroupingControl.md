# GroupingControl

## Назначение
Контрол `GroupingControl` отвечает за группировку входных данных по артикулам и отображение каждой группы в отдельной вкладке (`TabControl`). Он является родительским контейнером для множества `ArticleGroupingControl` (по одному на каждый артикул).

## Основные функции
1.  **Группировка данных:** Анализирует `DataTable` и группирует строки по ключевым столбцам (артикулам).
2.  **Управление вкладками:** Динамически создает `TabItem` для каждого уникального артикула, помещая в него `ArticleGroupingControl`. Строки с ошибками группируются в отдельную вкладку "Ошибки данных".
3.  **Управление настройками:** Хранит и предоставляет доступ к индивидуальным настройкам каждого артикула (`ArticleSettings`).
4.  **Валидация:** Не позволяет переключить вкладку, если на текущей вкладке есть неисправленные ошибки ручного раскроя (выводит `MessageBox`).
5.  **Визуализация:** Изменяет цвет вкладок в зависимости от того, есть ли у артикула индивидуальные (кастомные) настройки (отличные от настроек по умолчанию) и активна ли вкладка в данный момент.
6.  **Синхронизация UI:** Обеспечивает синхронизацию ширины левой панели (`LeftPanelWidth`) между всеми вложенными `ArticleGroupingControl`.

## Зависимости
- `Models.ArticleSettings`: Модель для хранения индивидуальных настроек артикула.
- `Models.PresetModel`: Модель настроек оборудования.
- `Models.StockLengthModel`: Модель длины хлыста со склада.
- `Controls.ArticleGroupingControl`: Вложенный контрол для детальной настройки и ручного раскроя конкретного артикула.
- `Services.CutSettingsProvider`: Провайдер настроек (используется для сохранения и загрузки ширины левой панели).
- `DataHelper`: Вспомогательный класс для формирования имени артикула из значений ключевых столбцов.

## Особенности реализации
- Реализует `INotifyPropertyChanged` для поддержки привязки данных (например, `LeftPanelWidth`).
- Использует событие `SelectionChanged` у `TabControl` для валидации перед сменой вкладки. При наличии ошибок ручного раскроя переключение отменяется.
- Вкладки перекрашиваются динамически (метод `RecolorAllTabs`):
    - Активная вкладка: Светло-голубая (по умолчанию) или Светло-коралловая (если есть кастомные настройки).
    - Неактивная вкладка: Белая (по умолчанию) или Бледно-розовая (если есть кастомные настройки).
- Вложенные `ArticleGroupingControl` создаются только один раз при вызове `RunGroupingWithTabs`.
