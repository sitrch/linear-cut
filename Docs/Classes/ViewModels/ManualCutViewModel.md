# ManualCutViewModel

## Назначение
Отвечает за бизнес-логику пользовательского интерфейса вкладки (или окна) "Ручной раскрой". Управляет списком строк ручного раскроя (`ManualCutRow`), контролирует наличие деталей, рассчитывает доступное место на хлысте с учетом толщины пропилов (`CutWidth`) и торцевания (`TrimStart`, `TrimEnd`), а также выполняет сложную валидацию ввода.

## Основные свойства и коллекции

- **`ObservableCollection<ManualCutRow> ManualCuts`**: Коллекция строк ручного раскроя. Привязана к DataGrid.
- **`AvailableSizes`**: Глобальный список доступных размеров деталей (тип `PartSizeAvailability`), используется для отображения состояния деталей.
- **`BarLength` / `BarLengths`**: Текущая длина хлыста и список доступных длин.
- **`CurrentPreset`**: Ссылка на текущие настройки (пресет), критически важна для расчетов пропилов.
- **`ErrorsText` / `HasErrorsText`**: Хранит и отображает агрегированные сообщения об ошибках валидации.

## Ключевые методы

- **`Initialize(...)`**: Первичная настройка ViewModel. Принимает данные из основного окна (список деталей, пресет).
- **`CalculateAvailableParts(...)`**: Заполняет внутренний словарь `_availableParts` общим количеством деталей, требующих раскроя.
- **`CalculateUsedParts()`** (private): Проходит по всем строкам `ManualCuts` и подсчитывает, сколько деталей каждого размера уже распределено. Записывает результат в `_usedParts`.
- **`PrepareAvailableSizesForDropdown(ManualCutRow row, int columnIndex)`**:
  Сложный метод, вызываемый при открытии ComboBox в UI (выбор размера детали в одной из 4-х колонок). Он рассчитывает оставшееся свободное место в *конкретном хлысте* (с учетом длины хлыста, торцеваний, уже выбранных деталей в других колонках этой же строки и толщины пропилов). Обновляет списки `AvailableSizes1..4` внутри строки, чтобы интерфейс мог подсветить красным размеры, которые не влезают.
- **`RefreshValidation()`**: Главный метод валидации. Вызывается при любых изменениях (добавление строки, изменение размера, количества).
  - Проверяет, не превышает ли сумма длин деталей + пропилов + торцеваний длину хлыста (`HasLengthError`).
  - Проверяет, не превышает ли количество использованных деталей конкретного размера общее доступное количество (`HasCountError`).
  - Проставляет соответствующие флаги ошибок (например, `HasLengthErrorSize1`), которые через Binding в XAML подсвечивают ячейки красным цветом.

## Команды
- **`AddRowCommand`**: Добавляет новую пустую строку `ManualCutRow` в конец списка.
- **`DeleteRowCommand`**: Удаляет выбранную строку из списка и вызывает пересчет валидации.

## Связь с UI
Взаимодействует с `ManualCutControl`. Для корректной работы требуется, чтобы события `CollectionChanged` и `PropertyChanged` внутри `ManualCutRow` своевременно вызывали `RefreshValidation()`.
