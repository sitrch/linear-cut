# RelayCommand

## Назначение
Реализация интерфейса `ICommand` для использования в паттерне MVVM. Позволяет привязывать действия пользователя в UI (например, нажатия кнопок) к методам ViewModel без необходимости создания отдельного класса для каждой команды.

## Основные компоненты и методы

- **`ICommand`**:
  Стандартный интерфейс WPF для реализации команд.

- **Поля `_execute` и `_canExecute`**:
  Хранят делегаты (указатели на методы), переданные из ViewModel. `_execute` (обязательный) выполняет само действие, `_canExecute` (необязательный) определяет, разрешено ли действие в данный момент.

- **Конструктор**:
  Принимает делегаты `execute` (Action<object>) и опционально `canExecute` (Predicate<object>).

- **`CanExecute(object parameter)`**:
  Вызывает делегат `_canExecute`. Если он не задан, возвращает `true` (команда доступна всегда).

- **`Execute(object parameter)`**:
  Вызывает делегат `_execute`, выполняя основную логику команды.

- **Событие `CanExecuteChanged`**:
  Особая реализация события. Она перенаправляет подписку на `CommandManager.RequerySuggested`. Это означает, что WPF будет автоматически опрашивать метод `CanExecute` при любых изменениях в UI (смена фокуса, ввод текста), автоматически включая или отключая элементы управления (например, кнопки).

## Зависимости
- `System.Windows.Input`: для `ICommand` и `CommandManager`.

## Особенности
Является стандартным паттерном для WPF MVVM приложений. Делегирует ответственность за выполнение команд и проверку их доступности обратно во ViewModel.
